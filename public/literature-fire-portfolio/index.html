智能建造技术 在建筑消防领域 中的应用 文献分析

我使用 Claude Code 工具创新设计了一种可交互的文献分析图，能够清晰地查看文献之间的关系，深受老师和同学们的赞扬。以下是我的项目介绍。

BIM / IoT / Digital Twin / AI CNKI Literature Mapping Claude Code / HTML / D3
01 / Project Context

这不是单纯做展示，而是先把文献逻辑梳理清楚。

课程任务要求我梳理 智能建造 技术如何进入建筑消防领域。前期我先确定研究范围，再围绕 BIM、物联网、数字孪生、AI、消防预警、风险评估等关键词检索文献，重点不是堆材料，而是看清不同研究之间的连接方式。

课程问题 智能建造技术在消防设计、施工、运维和应急管理中分别解决什么问题。
文献线索 BIM、AIoT、数字孪生、图像识别和传感网络在文献中反复出现。
分析重点 我更关心技术与应用场景之间的关系，而不是只统计关键词出现次数。
02 / My Workflow

传统文献分析图不够直观，我开始尝试用 HTML 重做。

我试过常见的文献分析方法，比如 CiteSpace 一类工具，但生成的图更适合分析过程，不太适合清晰展示。于是我改用 HTML 做交互图谱，边调结构、边改视觉、边检查逻辑。

01
检索文献

围绕 智能建造、BIM、物联网、人工智能、数字孪生、建筑火灾和风险预警等关键词，在 CNKI 中筛选课程任务相关文献。

CNKI关键词组合文献筛选
02
整理逻辑

把文献拆成引用关系、应用场景和技术演化三类问题，判断哪些研究互相连接，哪些方向正在聚集。

引用关系应用场景演化脉络
03
尝试 HTML 图谱

用 Claude Code 辅助编写 HTML、JavaScript 和 D3 逻辑，把文献节点、关键词、年份和关联强度转成可拖拽、可缩放的页面。

Claude CodeHTMLJavaScriptD3
04
反复调整

03 / Core Highlight

最后我把文献关系做成了可以操作的知识界面。

这部分是我在项目中投入最多的尝试。我没有停留在词云、柱状图或静态截图，而是用 Claude Code 辅助，把文献关系做成可搜索、可筛选、可拖拽、可缩放的交互式 HTML。这样展示时不只是给出结论，也能直接进入图谱观察文献之间的联系。

04 / What I Found

交互图谱帮助我把文献结论讲得更清楚。

01 / BIM BASE

BIM 仍然是建筑消防智能化的重要底座，常与空间建模、疏散模拟和运维管理连接。

02 / REAL-TIME SENSE

物联网、传感器和图像识别更多指向实时感知与早期预警。

03 / DIGITAL TWIN

数字孪生把模型、现场数据和预测算法连接起来，是一个明显的整合方向。

04 / CLOSED LOOP

整体趋势从单点技术应用，逐渐走向设计、施工、运维和应急处置的闭环管理。

Final / Design Value

这套 HTML 图谱，是一次课程任务里的方法尝试。

它把复杂文献关系从静态截图变成可以探索的页面。这个过程也让我意识到，文献综述不一定只能停留在文字和表格里；当关系足够复杂时，交互式网页可以成为一种更清晰的表达方式。
